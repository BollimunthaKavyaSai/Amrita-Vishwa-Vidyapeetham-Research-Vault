About this website

Amrita Research Paper Vault

A student‑first portal to discover campus research, browse domains, search papers, read updates, and collaborate via blogs and FAQs.

Overview

The vault centralizes student‑published research papers and provides fast search across titles, authors, domains, years, keywords, and DOIs. It also offers an activity layer with reviews, blogs, and FAQs for knowledge sharing.

Key features

Powerful search

Search by domain, keywords, year, author, or DOI with instant paging.

Domain browsing

Quick filters like AI, IoT, Cybersecurity, Data Science, and more.

Activity & community

Reviews on the homepage, blogs for updates, and FAQs for Q&A.

Personal dashboard

See the current profile, quick links, and recent activity in one place.

Contact integration

Contact form auto‑fills the logged‑in user’s name and email.

Secure auth

Password hashing and JWT‑based sessions with profile‑aware pages.

Getting started

  • Sign up on the signup page. A success message will ask to login.
  • Login to get redirected to the Dashboard with the current profile.
  • Use the navbar to jump to Search, Blog, FAQ, and Contact.

Daily workflow

  • Search or browse domains to discover papers.
  • Add reviews from the homepage review section.
  • Post updates in Blog and ask/answer in FAQs.

Data & Search

  • Papers are loaded from the campus dataset (CSV) and indexed in memory for rapid search.
  • The search endpoint returns paginated results with total matches for smooth browsing.
  • Common filters: domain, keywords, year, author names, and DOI substrings.

Accounts

  • Signup creates a user with role “user”; a toast asks to login.
  • Login stores the profile and token; Dashboard renders the current user.
  • Logout clears the token and redirects to the login page.

Security

  • Passwords are hashed; tokens expire and are verified on protected routes.
  • Contact can read email from the token if the field is left blank while logged in.

FAQs (quick)

  • Why can’t I comment or like? Ensure login; actions require an active session.
  • Where is my email on Contact? It auto‑fills from the logged‑in profile.
  • How do I view all papers? Use the “View All Papers” button or domain links.

Contact
